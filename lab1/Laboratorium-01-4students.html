Szyfr Cezara¶

Szyfr Cezara oczywicie nie zawdzicza swego istnienia Juliuszowi Cezarowi. Zosta przeze jedynie wspomniany w pamitnikach o wojnach galijskich. Prawdopodobnie bya to dobrze znana metoda w tym czasie.

Zastanów się:¶

1. Czy szyfr Cezara jest łatwy do złamania? Ile prób (testów przesunięć) trzeba aby złamać szyfr Cezara?¶
  • Potrzeba o jedn mniej ni liczba liter w alfabecie, przy pomocy ktrego zapisana zostaa wiadomo. Wynika to std, e w szyfrze Cezara kada litera zaszyfrowanej wiadomoci jest przesunita o tyle samo pozycji w alfabecie. Wystarczy wic sprawdzi wszystkie moliwe przesunicia, a jest ich o 1 mniej ni liczba liter w alfabecie ($k=0$ nie sprawdzamy, bo wtedy wiadomo jest taka sama jak szyfrogram).
2. Co to jest atak siłowy (brute force)?¶
  • Jest to atak, w ktrym sprawdzane s wszystkie moliwoci. W przypadku szyfru Cezara, atak brute force polegaby na sprawdzeniu wszystkich moliwych przesuni liter alfabetu.
3. Zaproponuj i zaimplementuj wariant szyfru Cezara, która wymaga wykonania większej liczby prób podczas ataku?¶
  • Szyfr Cezara z zastosowaniem klucza, okrelajcego rne przesunicie dla kolejnych liter szyfrowanej wiadomoci.

lub:

  • Szyfr Vinegere'a - wykorzystywana jest tablica kwadratowa, w ktrej kady wiersz zawiera kolejne litery alfabetu przesunite o inn liczb pozycji (w kadym kolejnym wierszu litery s przesunite o 1 pozycj w lewo wzgldem poprzedniego). Potrzebny jest klucz (tym razem klucz jest tekstem, zbudowanym przy pomocy tego samego alfabetu, co szyfrowana wiadomo) o dugoci rwnej dugoci szyfrowanej wiadomoci. Nastpnie, dla kolejnych par liter, stojcych na odpowiednio tych samych pozycjach szyfrowanej wiadomoci i klucza, wyznaczana jest litera z tablicy, jako przecicie kolumny, odpowiadajcej literze szyfrowanego tekstu oraz wiersza, otrzymanego dla danej litery z klucza.

Aby odszyfrowa wiadomo, trzeba znale wiersz, ktry odpowiada danej literze klucza, przy pomocy ktrego zaszyfrowana zostaa wiadomo. Nastpnie, w tym wierszu naley znale pooenie litery z zaszyfrowanej wiadomoci, stojcej na tej samej pozycji, co litera z klucza, odpowiadajca danemu wierszowi. Litera przypisana kolumnie, w ktrej znajduje si znaleziona litera z zaszyfrowanej wiadomoci, odpowiada literze z wyjciowej wiadomoci (tej, ktra zostaa zaszyfrowana).

4. Oceń koszt ataku na ten wariant Cezara.¶

Szyfr Cezara z kluczem okrelajcym rne przesunicia liter:

  • W przypadku pierwszego wariantu, w ktrym korzystamy tylko z rnego przesunicia dla kolejnych liter alfabetu, konieczne bdzie wykonanie $L^n - 1$ prb, gdzie $L$ - jest dugoci alfabetu, a $n$ - dugoci szyfrowanej wiadomoci.

Szyfr Vinegere'a:

  • Koszt jest taki sam, jak w poprzednim algorytmie, poniewa konieczne jest sprawdzenie wszystkich moliwych kluczy, a na kadej pozycji klucza moe znajdowa si jedna z liter alfabetu.

alphabet = "abcdefghijklmnopqrstuvwxyz"

def caesar_init(alphabet):
    positions = { k: i for i, k in enumerate(alphabet) }
    
    def encrypt(text, key):
        result = []
        for i, char in enumerate(text):
            offset = key[i % len(key)]
            if char in positions:
                result.append(alphabet[(positions[char] + offset) % len(alphabet)])
            else:
                result.append(char)
        return ''.join(result)
            
    def decrypt(encrypted_text, key):
        result = []
        for i, char in enumerate(encrypted_text):
            offset = key[i % len(key)]
            if char in positions:
                result.append(alphabet[positions[char] - offset])
            else:
                result.append(char)
        return ''.join(result)
        
    return encrypt, decrypt

encrypt, decrypt = caesar_init(alphabet)

import numpy as np

alphabet = "abcdefghijklmnopqrstuvwxyz"

def permutation_caesar_init(alphabet):
    secret_alphabet = ''.join(np.random.permutation(list(alphabet)))
    positions = { k: i for i, k in enumerate(secret_alphabet) }
    
    def encrypt(text, key):
        return ''.join([secret_alphabet[(positions[char] + key) % len(alphabet)] 
                        if char in positions else char for char in text])
            
    def decrypt(encrypted_text, key):
        return ''.join([secret_alphabet[(positions[char] - key)] 
                        if char in positions else char for char in encrypted_text])
    
    return encrypt, decrypt

Kryptoanaliza szyfrów monoalfabetycznych¶

Wspczenie kryptoanaliza szyfru Cezara jest trywialna. Mona po prostu sprawdzi wszystkie 26 moliwych przesuni.

Szyfry monoalfabetyczne nie s take odporne na kryptoanaliz statystyczn. Znajc rozkad czstoci liter w tekstach napisanych w danym jzyku moemy odgadn przesunicie obserwujc minima i maxima w histogramach.

W zasadzie kluczow spraw w obu przypadkach jest znajomo jzyka, a w gruncie rzeczy alfabetu.

    if (char not in unique) and (char.isalpha()):
        unique.append(char)

for unique_char in unique: #czsto liter w tekcie 
     frequency.append(ciphertext.count(unique_char) / float(len(only_alphabets)))

         value = value + frequency[unique.index(unique_char)] * english_frequency_prob[alphabets.index(unique_char) - i]

    decoded_string = ""

Zastanów się:¶

  1. Sprawd jak wyglda histogram szyfrogramu.
  • Inaczej ni histogram szyfrowanej wiadomoci. Jego wygld jest zaleny od uytego algorytmu szyfrowania.
  1. Zastanw si jakie znaczenia ma dugo klucza dla bezpieczestwa szyfrowania.
  • Im duszy klucz, tym zaszyfrowana wiadomo jest trudniejsza do odszyfrowania.
  1. Czy powtarzanie klucza mona wykorzysta dla potrzeb kryptoanalizy?
  • Wielokrotne wykorzystywanie tego samego klucza moe dawa w rezultacie szyfrogramy o podobnych cechach. Z rego powodu, mona wykorzysta powtarzanie klucza dla potrzeb kryptoanalizy, w celu sprawdzenia jakoci algorytmu szyfrowania.
  1. Jaki klucz gwarantowaby bezpieczestwo?
  • Taki klucz, ktry nie zosta wczeniej wykorzystany i jego dugo jest przynajmniej taka jak dugo szyfrowanej wiadomoci.
  1. Jaki rozkad czstoci symboli szyfrogramu nie ujawni adnych przydatnych danych kryptoanalitykowi?
  • Rozkad, w ktrym czsto wystpowania poszczeglnych symboli jest niezalena od czstoci wykorzystywania tych symboli w sowach danego jzyka. Wwczas, nie mona dopasowa liter szyfrogramu do odpowiednich liter alfabetu szyfrowego, na podstawie czstoci ich wystpowania.

Szyfr Enigma¶

Opis¶

  • Enigma jest urzdzeniem przypominajcym maszyn do pisania, skadajcym si z nastpujcych elementw, biorcych udzia w szyfrowaniu wiadomoci:
    • klawiatura,
    • panel (plugboard), pozwalajcy na rczne przepicie par liter (np. zamian litery A z liter G),
    • rotory (wirniki) (pierwotnie 3, pniej nawet 7), zmieniajce swoje uoenie po kadym naciniciu przycisku na klawiaturze,
    • reflektor (dziki niemu sygna wraca z powrotem przez obwd do klawiatury i skutkuje zapaleniem si odpowiedniej lampki, oznaczajcej zakodowany znak)
  • Po naciniciu dowolnego klawisza na klawiaturze, obraca sipierwszy w kolejnoci rotor. Kady rotor posiada wgbienie. Po obrceniu si pierwszego rotora do pozycji, w ktrej znajduje si wgbienie ssiadujcego rotora, nastpuje obrt obu rotorw rwnoczenie. Zakadajc, e alfabet szyfrowy ma $26$ znakw, moemy zauway, i pierwszy rotor obraca si za kadym razem, gdy naciniemy klawisz na klawiaturze. Kolejny rotor obraca si co $26$ wprowadzonych znakw, nastpny co $26 \cdot 26 = 676$, itd.
  • kady z rotorw posiada zdefiniowane mapowanie liter alfabetu na pewnpermutacj tych liter. Przykadowo, pierwszy rotor moe przeksztaca kolejne litery alfabetu w nastpujcy sposb: ABCDEFGHIJKLMNOPQRSTUVWXYZ -> EKMFLGDQVZNTOWYHXUSPAIBRCJ,
  • kady rotor posiada rwnie ustalon pozycj wgbienia, powodujcego jego obrt,
Szyfrowanie wiadomości¶
  • na pocztku naley wybra konfiguracj maszyny (ustawi poczenia na panelu i dobra rotory, ktre maj odpowiednie mapowania liter oraz pozycje wgbie),
  • szyfrowanie odbywa si poprzez wpisywanie kolejnych liter na klawiaturze,
  • jako klucz szyfrowania suy pocztkowa konfiguracja maszyny (uoenie i rodzaj rotorw oraz poczenia w panelu)
Odszyfrowywanie wiadomości¶
  • aby odszyfrowa wiadomo, naley dysponowatakim samym zestawem rotorw (uoone w takiej samej kolejnoci, z takim samym mapowaniem liter alfabetu i tymi samymi pozycjami wgbie),
  • konieczne jest rwnie podczenie liter na panelu w taki sam sposb jak w maszynie, na ktrej zostaa zaszyfrowana wiadomo,
  • rotory naley obrci w taki sposb, jak na pocztku w maszynie, na ktrej odbywao si szyfrowanie (ustawi na nich takie same pocztkowe symbole (litery)),
  • po prawidowej konfiguracji pocztkowej, wystarczy wpisa otrzymany szyfrogram, eby otrzyma tekst jawny

'''
Rotor wiring (Enigma I): https://www.wikiwand.com/en/Enigma_rotor_details
'''

class Keyboard:
  def __init__(self, alphabet):
    self.letters = alphabet
    self.signals = {l: i for i, l in enumerate(alphabet)}

  def forward(self, letter):
    return self.signals[letter]

  def backward(self, signal):
    return self.letters[signal]

class Component:
  def __init__(self, left, right):
    self.left = left
    self.right = right

  def forward(self, signal):
    letter = self.right[signal]
    return self.left.index(letter)

  def backward(self, signal):
    letter = self.left[signal]
    return self.right.index(letter)

class Plugboard(Component):
  def __init__(self, alphabet, pairs):
    left = list(alphabet)
    right = list(alphabet)

    positions = {l: i for i, l in enumerate(alphabet)}

    for l1, l2 in pairs:
      idx1 = positions[l1]
      idx2 = positions[l2]
      left[idx1], left[idx2] = left[idx2], left[idx1]

    super().__init__(left, right)
    
class Rotor(Component):
  def __init__(self, alphabet, wiring, notch):
    super().__init__(alphabet, wiring)
    self.alphabet = alphabet
    self.notch = notch

  def rotate(self, steps=1, forward=True):
    if forward:
      self.left = self.left[steps:] + self.left[:steps]
      self.right = self.right[steps:] + self.right[:steps]
    else:
      self.left = self.left[-steps:] + self.left[:-steps]
      self.right = self.right[-steps:] + self.right[:-steps]

  def rotate_to_letter(self, letter):
    self.rotate(self.left.index(letter))

  def set_ring(self, position):
    # rotate the rotor backwards
    self.rotate(position, forward=False)
    # adjust the notch position
    notch_position = self.alphabet.index(self.notch)
    self.notch = alphabet[notch_position - position]

class Reflector(Component):
  def __init__(self, alphabet, wiring):
    super().__init__(alphabet, wiring)

  def reflect(self, signal):
    return super().forward(signal)

  # Remove forward and backward methods from the Reflector class
  def forward(self, *args, **kwargs):
    raise AttributeError(f"'{self.__class__.__name__}' object has no attribute 'forward'")

  def backward(self, *args, **kwargs):
    raise AttributeError(f"'{self.__class__.__name__}' object has no attribute 'backward'")

class Enigma:
  def __init__(self, keyboard, plugboard, rotors, reflector):
    self.keyboard = keyboard
    self.plugboard = plugboard
    self.rotors = rotors
    self.reflector = reflector

  def set_key(self, key):
    if len(key) != len(self.rotors):
      raise AttributeError(f"'key' must have {len(self.rotors)} letters")
    for i, letter in enumerate(key):
      self.rotors[i].rotate_to_letter(letter.upper())

  def set_rings(self, rings):
    if len(rings) != len(self.rotors):
      raise AttributeError(f"'rings' must contain {len(self.rotors)} numbers")
    for i, ring in enumerate(rings):
      self.rotors[i].set_ring(ring)

  def encrypt(self, message):
    return ''.join([self.__encrypt_letter(letter) for letter in message.upper()])
    
  def decrypt(self, cipher):
    # Decryption is the same as encryption
    return ''.join([self.__encrypt_letter(letter) for letter in cipher.upper()])

  def __encrypt_letter(self, letter):
    self.__rotate_rotors()
    signal = self.keyboard.forward(letter)
    signal = self.plugboard.forward(signal)
    for rotor in self.rotors[::-1]:
      signal = rotor.forward(signal)
    signal = self.reflector.reflect(signal)
    for rotor in self.rotors:
      signal = rotor.backward(signal)
    signal = self.plugboard.backward(signal)
    return self.keyboard.backward(signal)

  def __rotate_rotors(self):
    for rotor in self.rotors[::-1]:
      rotate_next = rotor.left[0] == rotor.notch
      rotor.rotate()
      if not rotate_next:
        return

Szyfr Playfair¶

Opis¶
  • Szyfr Playfair polega na utworzeniu tablicy znakw o wymiarach $5 \times 5$, a nastpnie, przeksztacaniu kolejnych par liter tekstu jawnego w okrelony sposb, korzystajc z tej tablicy.
Tworzenie tablicy¶
  • Do stworzenia tablicy jest wykorzystywany podany klucz szyfrowania oraz litery alfabetu szyfrowego,
  • Tablica jest wykorzystywana zarwno przy szyfrowaniu jak i przy odszyfrowywaniu,
  • W pierwszym kroku, tworzona jest tablica $5 \times 5$, w nastpujcy sposb:
    • jako kolejne litery w wierszu wpisywane s kolejne litery klucza z pominiciem tych, ktre wystpiy wczeniej,
    • litery $i$ oraz $j$ s traktowane tosamo (jako takie same litery, poniewa jest 26 liter alfabetu aciskiego, a tablica ma tylko 25 pl, wic trzeba pozby si jednej z liter. Zazwyczaj pozbywamy silitery $i$ lub $j$),
    • jeeli do tablicy wpisalimy liter$i$, nie wpisujemy ju litery $j$ (lub, jeeli wpisalimy liter$j$, nie wpisujemy litery $i$),
    • nastpnie, tablic dopeniamy pozostaymi literami alfabetu w takiej kolejnoci, w jakiej te litery wystpujw alfabecie (trzymamy si rwnie wyej opisanych zasad, wic pomijamy wpisane wczeniej litery i traktujemy litery $i$ oraz $j$ jako t sam liter),

Szyfrowanie¶
  • W kolejnym kroku, dzielimy tekst jawny na bloki, skadajce siz 2 liter, zgodnie z poniszymi zasadami:
    • 2 ssiednie litery tworz pojedynczy blok,
    • jeeli obie litery s takie same, zamieniamy 2. liter na inn (zwykle przyjmuje si, e jest to litera $x$),
    • jeeli ostatni blok skada siz 1 litery (bo tekst jawny ma nieparzyst liczbznakw), dopisujemy do tego bloku liter $x$, tworzc par liter,
  • Szyfrujemy kolejne pary, zgodnie z poniszymi zasadami:
    • jeeli obie litery z pary znajdujsiw tym samym wierszu, zastpujemy je przed ich ssiadw, lecych po ich prawej stronie. Jeeli jedna z liter znajduje si w ostatniej kolumnie, zastpujemy jprzez liter z tego samego wiersza, lecw pierwszej kolumnie,
    • jeeli obie litery znajduj si w tej samej kolumnie, zastpujemy je przez ich ssiadw, lecych jeden wiersz niej wzgldem odpowiednich liter. Jeeli jedna z liter znajduje si w ostatnim wierszu, zastpujemy jprzez literz pierwszego wiersza (lec w tej samej kolumnie),
    • jeeli litery nie le ani w tym samym wierszu, ani w tej samej kolumnie, kad z liter z pary zastpujemy liter, znajdujc si w tym samym wierszu, lec w tej kolumnie, w ktrej znajduje sidruga litera z pary,
  • Tak otrzymane pary czymy w jeden cig tekstowy i otrzymujemy szyfrogram.
Odszyfrowywanie¶
  • Zaczynamy tak samo, jak przy szyfrowaniu, dzielc szyfrogram na 2-literowe bloki,
  • Wykonujemy operacje odwrotne, ni przy szyfrowaniu:
    • jeeli litery znajduj si w tym samym wierszu, zastpujemy je odpowiednio przez ich ssiadw z lewej strony (pamitajc, e gdy jedna z liter jest w pierwszej kolumnie, bierzemy liter z ostatniej kolumny),
    • jeeli litery s w tej samej kolumnie, zastpujemy je literami, ktre znajduj si 1 wiersz nad nimi (jak jedna z liter jest w pierwszym wierszu, bierzemy literz ostatniego wiersza),
    • jeeli litery z pary nie s ani w jednym wierszu ani w jednej kolumnie, postpujemy identycznie jak przy szyfrowaniu,
  • Otrzymane pary czymy w cig tekstowy, otrzymujc tekst jawny.

class Playfair:
  alphabet = "abcdefghijklmnopqrstuvwxyz"

  def __init__(self, key):
    self.table = [[''] * 5 for _ in range(5)]
    self.key = key.lower()
    self.__fill_table()
    self.coordinates = self.__get_coordinates()

  def encrypt(self, message):
    # Divide the message into pairs of 2 different letters
    pairs = self.__split_into_pairs(message)
    return ''.join(self.__encrypt_pair(pair) for pair in pairs)

  def decrypt(self, cipher):
    pairs = self.__split_into_pairs(cipher)
    return ''.join(self.__decrypt_pair(pair) for pair in pairs)

  def __split_into_pairs(self, message):
    pairs = []
    for i in range(0, len(message), 2):
      left = message[i]
      right = message[i + 1] if i < len(message) - 1 and message[i + 1] != message[i] else 'x'
      pairs.append(left + right)
    return pairs

  def __fill_table(self):
    used_letters = set()

    def is_used(letter):
      return letter in used_letters \
          or letter == 'i' and 'j' in used_letters \
          or letter == 'j' and 'i' in used_letters

    def put_letter(letter, idx):
      self.table[idx // 5][idx % 5] = letter
      used_letters.add(letter)
    
    i = 0
    for letter in self.key:
      if is_used(letter): continue
      put_letter(letter, i)
      i += 1
    
    for letter in self.alphabet:
      if is_used(letter): continue
      put_letter(letter, i)
      i += 1

  def __get_coordinates(self):
    coordinates = {}

    for i in range(5):
      for j in range(5):
        coordinates[self.table[i][j]] = (i, j)

    coordinates['j'] = coordinates['i']
    coordinates['i'] = coordinates['j']

    return coordinates

  def __encrypt_pair(self, pair):
    r1, c1 = self.coordinates[pair[0]]
    r2, c2 = self.coordinates[pair[1]]

    # If the letters appear on the same row of your table, replace them with the letters to their immediate right respectively (wrapping around to the left side of the row if a letter in the original pair was on the right side of the row)
    if r1 == r2:
      return self.table[r1][(c1 + 1) % 5] + self.table[r1][(c2 + 1) % 5]
    # If the letters appear on the same column of your table, replace them with the letters immediately below respectively (wrapping around to the top side of the column if a letter in the original pair was on the bottom side of the column)
    if c1 == c2:
      return self.table[(r1 + 1) % 5][c1] + self.table[(r2 + 1) % 5][c1]
    # If the letters are not on the same row or column, replace them with the letters on the same row respectively but at the other pair of corners of the rectangle defined by the original pair.
    return self.table[r1][c2] + self.table[r2][c1]

  def __decrypt_pair(self, pair):
    r1, c1 = self.coordinates[pair[0]]
    r2, c2 = self.coordinates[pair[1]]

    if r1 == r2:
      return self.table[r1][(c1 - 1) % 5] + self.table[r1][(c2 - 1) % 5]
    if c1 == c2:
      return self.table[(r1 - 1) % 5][c1] + self.table[(r2 - 1) % 5][c1]
    return self.table[r1][c2] + self.table[r2][c1]

Szyfr Lorenza¶

Opis¶
  • Szyfr Lorenza jest trudniejszy do zamania ni szyfr Enigma,
  • Maszyna Lorenza jest zbudowana, podobnie do Enigmy, z wirnikw, ktre dziaaj w inny niw Enigmie sposb,
Szyfrowanie¶
  • Kada z liter, znajdujcych sina klawiaturze, bya przeksztacana na odpowiadajcy jej cig bitowy (najczciej kod Baudot, skadajcy si z 5 bitw),
  • Wprowadzenie litery powodowao wygenerowaniu pseudolosowego cigu bitw (jeeli litery kodujemy 5-bitowym cigiem Baudot, otrzymywalimy 5-bitowy cig pseudolosowych bitw). Cig ten by generowany poprzez obrt 5 wirnikw (rotorw),
  • Nastpnie, wykorzystywano szyfrowanie Vernama, czyli wykonywano operacjXOR na 5-bitowym cigu, odpowiadajcym kodowanej literze z tekstu jawnego oraz 5-bitowym cigu pseudolosowym,
  • Tak otrzymany cig bitw, zamieniano z powrotem na liter, ktrej on odpowiada (zgodnie z wykorzystywanym kodowaniem, np. kodowaniem Baudot), wszystkie zaszyfrowane litery czono i otrzymywano szyfrogram,
Odszyfrowywanie¶
  • Odszyfrowywanie dziaao identycznie jak szyfrowanie (wykonanie operacji XOR na kolejnych cigach 5-bitowych, z wykorzystaniem tych samych cigw pseudolosowych, co podczas szyfrowania, skutkowao otrzymaniem cigw 5-bitowych z tekstu jawnego),
  • Aby mc odszyfrowa wiadomo, trzeba byo posiada taksamkonfiguracjpocztkowmaszyny Lorenza, a wic mie taki sam zestaw tak samo ustawionych wirnikw. Takie same ustawienia pocztkowe gwarantoway, e maszyna wygeneruje ten sam cig liczb pseudolosowych, co pozwoli na odszyfrowanie wiadomoci.
Opis implementacji¶
  • Umieszczona niej implementacja jest implementacjuproszczon, pomijajc dziaanie generatora liczb pseudolosowych w maszynie Lorenza. Do wygenerowania kolejnych liczb pseudolosowych, wykorzystaem bibliotek random, ustawiajc taki sam seed (ziarno) podczas szyfrowania i odszyfrowywania wiadomoci,
  • Ustawienie ziarna odpowiada ustawieniu takiej samej konfiguracji maszyny przed rozpoczciem odszyfrowywania jak konfiguracja, jak mielimy przed rozpoczciem szyfrowania.

import random

class Lorenz:
  # Baudot-Murray codes
  codes = { 
    'a': '00011', 
    'b': '11001',
    'c': '01110',
    'd': '01001',
    'e': '00001',
    'f': '01101',
    'g': '11010',
    'h': '10100',
    'i': '00110',
    'j': '01011',
    'k': '01111',
    'l': '10010',
    'm': '11100',
    'n': '01100',
    'o': '11000',
    'p': '10110',
    'q': '10111',
    'r': '01010',
    's': '00101',
    't': '10000',
    'u': '00111',
    'v': '11110',
    'w': '10011',
    'x': '11101',
    'y': '10101',
    'z': '10001',
    ' ': '00100',
    'N': '00000',
    'C': '01000',
    'L': '00010',
    'F': '11011',
    'T': '11111'
  }

  def __init__(self):
    self.random = random.Random(0)

  def set_key(self, key):
    self.random = random.Random(key)

  def encrypt(self, message):
    return self.__run(message)

  def decrypt(self, cipher):
    return self.__run(cipher)

  def __run(self, text):
    # The Lorenz machine used a complicated system of rotors to generate
    # subsequent pseudorandom numbers. To simulate this behavior, we can
    # simply use the Python random function which will generate a pseudorandom
    # number and choose the corresponding character used as a bit stream to
    # encode a particular letter from the message to encrypt
    characters = list(self.codes.keys())
    binary = list(self.codes.values())
    key_stream = ''.join(self.random.choice(characters) for _ in range(len(message)))
    indices = [int(self.codes[letter], 2) ^ int(self.codes[key], 2) for letter, key in zip(text, key_stream)]
    return ''.join(characters[binary.index(f"{i:#07b}"[2:])] for i in indices)
